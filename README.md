## 项目开发文档：
### (一) 实现功能点：<br/>
**1. 基础功能点：**<br/>
 - 游戏各级界面，均有暂停/恢复/退出功能<br/>
 - 随机产生方块，且有方块移动动画<br/>
 - 多种不同类型的方块（6种基础方块，4消方块，5消方块，冰块）<br/>
 - 正常3消，消除一行/列的4消，消除多行/列的5消<br/>
 - 关卡得分统计及步数限制<br/>
 - 5个关卡，进入时方块随机<br/>
    
**2. 创新功能点**<br/>
 - 音乐和音效滑块控制<br/>
 - 除第0关外，各关均有剩余步数奖励机制<br/>
 - 商店功能，可利用金币购买特效<br/>
 - 皮肤机制，可以随意换肤，同时更换音效<br/>
 - 3种通关条件（无限制、消除指定方块、消除冰块）<br/>
 - 3种特殊道具（加3步、交换两个、重新洗牌）<br/>
    
### (二) 技术难点及解决方案：<br/>
1. 游戏的音量控制<br/>
         项目初期，在界面设计时遇到的第一个难点是声音控制。为了实现移动滑块以调节声音的效果，在一天内尝试了多种方案和触控器，无意之间还学会了分数栏/      加载栏的绿色逐渐加满的UI特效。最终，找到了cocos拓展当中的Slider对象(如下)，不用反复update，就可以实现控制效果。<br/>
2. 游戏逻辑中点击事件与消除动画的异步和延迟<br/>
         项目中期，游戏逻辑中遇到的最大的难点是动画与操作的不同步。由于场景需要不断刷新，而方块移动动画又有一定的延迟，出现了多次消除时画面混乱的情        况。起初个人一味的修改代码逻辑，而没有意识到本质问题。研究了一段时间后，决定将点击事件和消除逻辑设置不同的刷新器，同时设置好一定的刷新间隔，解决      了两者异步和延迟带来的问题。<br/>
3. 更换皮肤时图片类型的修改<br/>
项目后期，创新设计中遇到的难点是皮肤功能的引入。为了可以更换方块的皮肤，对先前设计的Block类进行了较大的修改。原先设计的Block类是虚基类，不同图片对应不同的Block的子类。为加入皮肤功能，将图片设为MenuItemImage的指针，同时为图片重新命名，使用不同的数字后缀,同时使用UserDefault来记录皮肤，将所有的基础方块合并为了一个类。<br/>
